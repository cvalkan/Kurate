**Key Technical Concepts**
-   **Tournament Stopping Criterion**: The session involved a deep dive into convergence metrics. It moved from a flawed Spearman's Rho (ρ) approach, to a theoretically better but practically problematic Bradley-Terry Confidence Interval (BT-CI) approach, and finally settled on reverting to the robust and proven **Wilson Confidence Interval** method. This ensures per-paper convergence and correctly handles papers with extreme win rates.
-   **Cloudflare Bypassing Strategy**: The agent systematically debugged multiple web scraping and automation tools (, , , ) to access ChemRxiv, ultimately discovering that the PDFs were mirrored on a non-Cloudflare-protected Cambridge University API. This highlights the importance of finding alternative data sources over fighting anti-bot measures directly.
-   **Authentication Flow**: Switched from  session cookies, which were failing due to cross-origin browser policies, to a **JWT token-based system using localStorage**. This is a standard and more reliable pattern for single-page applications.
-   **MongoDB Key Sanitization**: Diagnosed a subtle but critical bug where dots () in model names used as MongoDB field keys (e.g., ) were treated as path separators, corrupting the data structure. The fix involved sanitizing keys by replacing dots with underscores.
-   **Decoupled Admin Controls**: Refactored the scheduler and admin UI to allow for separate control over the Paper Fetching and Tournament processes, giving the admin more granular management capabilities.
