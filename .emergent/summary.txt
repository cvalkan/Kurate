**Key Technical Concepts**
-   **Surprisingly Popular (SP) Signal**: The difference between a paper's rank from a pairwise tournament (BT rank) and its rank from standalone scoring (SI rank). Found to be a strong quality predictor on datasets with a *comparative* ground truth.
-   **Comparative vs. Standalone Ground Truth**: A key insight that the best AI evaluation method (pairwise vs. single-item) depends on the cognitive process used to generate the human ground truth.
-   **Controlled Comparisons**: Ensuring experiments are fair by controlling for variables like the specific paper pairs and judge models used.
-   **Non-blocking Background Tasks**: The rating generation task was refactored to run in small, non-blocking batches using  to prevent server saturation.
-   **Dynamic UI Configuration**: The leaderboard fetches admin settings from the backend to conditionally render columns, allowing for dynamic UI changes without a code deployment.

**Critical Info for New Agent**
1.  **Rating Generation is Non-Blocking**: The task to generate ratings for existing papers runs in the background in small batches to keep the server responsive. It can be started and stopped from the admin panel.
2.  **Two Sources for Ratings**: A paper's  can come from (1) being automatically parsed from a new summary or (2) being generated after-the-fact for an old summary. The UI handles both cases.
3.  **The Gap Signal is Context-Dependent**: The Gap score is the Surprisingly Popular signal. Analysis showed this signal is most meaningful on datasets with a *comparative* ground truth (like ICLR).
4.  **Admin Toggles Control Leaderboard Columns**: The Rating and Gap columns are controlled by settings in the Admin panel. If they don't appear, check there first.
