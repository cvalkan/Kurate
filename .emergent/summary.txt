**Key Technical Concepts**
-   **DB-Backed Architecture**: The primary theme of the session. Migrating from a monolithic, memory-intensive in-memory cache to a scalable database-first model for the leaderboard, enabling the system to handle much larger data volumes.
-   **OOM Kill Diagnosis & Prevention**: Using a custom-built memory logging system to capture RSS usage at critical code paths, which allowed for precise identification and fixing of memory spikes caused by concurrent background tasks.
-   **Incremental Updates & Reconciliation**: A robust data management pattern where live data is updated incrementally on events (e.g., match completion), with a periodic reconciliation job to verify consistency and correct any drift, ensuring both performance and correctness.
-   **Keyset (Cursor-based) Pagination**: The implementation of efficient, O(1) pagination for the API, crucial for scalable infinite-scrolling interfaces. This was contrasted with the performance degradation of traditional offset/limit pagination at scale.
-   **Lazy/Asynchronous Initialization**: Refactoring blocking I/O operations (like font cache generation) that were at the module-import level into asynchronous, lazily-executed tasks to prevent them from hanging or crashing the server startup process.
