<analysis>**original_problem_statement:**
The user wants to create a web application that automatically fetches academic papers from arXiv, ranks them using pairwise comparisons from multiple Large Language Models (LLMs), and displays the results on a public-facing leaderboard.

**PRODUCT REQUIREMENTS:**
- **Paper Fetching:** Automatically fetch papers from specified arXiv categories.
- **LLM-based Ranking:** Use multiple LLMs (GPT, Claude, Gemini) to compare pairs of papers based on their full text or abstracts.
- **Ranking Algorithm:** Implement a Bradley-Terry model to calculate Elo-like scores from pairwise comparisons.
- **Adaptive Matchmaking:** Develop a sophisticated scheduler to intelligently select which papers to compare next, focusing on papers that need it most to improve ranking efficiency.
- **Multi-Category Support:** The application must support multiple, independent tournaments for different arXiv categories, with no data cross-contamination.
- **Admin Panel:** A comprehensive admin dashboard to:
    - Monitor scheduler status and performance for each category.
    - View detailed statistics (paper counts, match counts, costs).
    - Manage and edit the prompts used for LLM comparisons.
    - Run experiments.
- **Public Leaderboard:** A user-friendly interface to display the rankings for each category, with features like:
    - Cross-category filtering using tags (with AND/OR logic).
    - Keyword search through paper titles.
    - Mobile-responsive design.
- **Model Analysis:** Provide analytics on the LLMs themselves, including:
    - Rank correlation (e.g., Spearman's rho) between different models.
    - Pairwise agreement rates, with a breakdown for clear-cut vs. contested pairs.
- **Surprisingly Popular Experiment:** Implement a feature to run separate tournaments to identify hidden gems by comparing a standard full-text ranking against a prediction-based ranking (where the LLM predicts which paper the crowd would find more impactful). This should support both abstract-only and full-text prediction modes.
- **Performance:** The application, especially the leaderboard, must be fast and responsive, even with a large number of papers and categories.

**User's preferred language**: English

**what currently exists?**
A full-stack application with a FastAPI backend and React frontend. It successfully fetches and ranks papers from multiple arXiv categories in separate, parallel tournaments. The backend features an adaptive matchmaking scheduler and a background cache that makes API responses nearly instant. The frontend provides a public leaderboard with multi-category views, tag-based filtering (AND/OR), and keyword search. A comprehensive admin panel allows for monitoring, prompt management, and running a Surprisingly Popular experiment with side-by-side comparison tables. The system also includes detailed model correlation and agreement analysis. All major user requests and bug fixes from the session have been implemented.
