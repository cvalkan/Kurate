**what currently exists?**
The project is a web application for ranking academic papers using an LLM-based judge. It features a Python/FastAPI backend with a MongoDB database and a React frontend.

Key implemented features include:
- A main leaderboard with filtering by primary category and time periods.
- A tag-based filtering system with a Global/Local statistics toggle.
- A robust user authentication system supporting email/password and Google OAuth, with mandatory email verification for the former.
- Gated content, where non-logged-in users have restricted access.
- An admin panel with functionality to manage users (view, deactivate/reactivate), view user suggestions, fetch new papers, and manage tournaments (view status, pause/resume).
- A background scheduler that automatically manages paper fetching and runs comparison tournaments for all active categories until predefined goals are met.
- A tournament registry in the database that acts as the single source of truth for all tournament states, which was implemented to improve scalability and control.
- A piggyback mechanism that records match data for secondary categories at no additional LLM cost.
- A mobile-responsive UI with a hamburger menu.
