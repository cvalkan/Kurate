<analysis>**original_problem_statement:**
The user wants to build a new application based on a previous project, PaperSumo. The new app, SciRank, is an automated platform that downloads new academic papers from arXiv for specific categories, runs a pairwise comparison tournament using multiple Large Language Models (LLMs), and displays a dynamically updated leaderboard.

**PRODUCT REQUIREMENTS:**
- **Automated Paper Fetching:** Automatically download new papers daily from specified arXiv categories (initially Robotics, later expanded to Distributed Computing, Economics, and Computational Physics).
- **LLM Tournament:** Randomly use one of three LLMs (GPT 5.2, Claude Opus 4.5, Gemini 3) to compare pairs of papers based on their full text.
- **Dynamic Leaderboard:** Display a public, ranked list of papers for each category.
- **Leaderboard Details:** The leaderboard must show paper title, authors, rank, score (initially Bradley-Terry, later Elo), win rate percentage, confidence interval (initially Elo-based, later Wilson Score Interval), number of matches, and publication date.
- **Filtering:** The leaderboard should be filterable by time periods (Most Recent, This Week, This Month, All Time).
- **Adaptive Matchmaking:** The system should intelligently select which papers to compare to balance ranking accuracy, speed of convergence, and fairness, especially for new papers.
- **Admin Panel:** A password-protected admin section is required for managing settings, viewing progress, and triggering manual actions.
- **Scientific Impact Summaries:** For papers that have achieved a stable ranking (settled), the system should generate a structured AI Impact Assessment using an LLM, based on the paper's content and its performance in the tournament.
- **Model Analysis:** A separate page to provide a correlation analysis between the results from the different LLMs.
- **Methodology Page:** A static page explaining how the platform works.
- **Multi-Category Support:** The entire platform (leaderboard, admin, backend processes) must support multiple, independent tournaments for different academic categories.

**User's preferred language**: English
