<analysis>**original_problem_statement:**
Build a website/platform that fetches the latest scientific papers from arxiv (sorted by different topic) and then runs a pairwise tournament on them to output a ranked list (e.g. based on Bradley-Terry scores) of scientific impact.

For that matter, it uses an LLM to determine which of the two selected papers has the higher estimated scientific impact or value. It can run multiple agents in parallel to speed up the process.

**PRODUCT REQUIREMENTS:**
- Use GPT 5.2 with the Emergent LLM key.
- Fetch papers from all arXiv categories.
- Tournament settings should be configurable.
- Results of tournaments must be preserved.
- Implement a Deep Analysis mode that reads full papers, not just abstracts.
- Add a feature to search for papers by keywords, authors, etc., to create custom tournament sets.
- Show the LLM's reasoning for each comparison in the tournament logs.
- Implement a UCB (Upper Confidence Bound) bandit mechanism to improve ranking efficiency.
- Lift the 20-paper limit on tournaments.

**what currently exists?**
A full-stack application (React + FastAPI) called PaperSumo that allows users to rank arXiv scientific papers.

**Key Features Implemented:**
- Fetches papers from arXiv by category.
- Advanced search to find papers by keywords, authors, and category, with support for exact phrase matching.
- Users can select papers to create a custom tournament.
- Two ranking modes for tournaments:
    1.  **Abstract-only:** A quick comparison based on paper abstracts.
    2.  **Deep Analysis:** A slower, more thorough comparison that downloads the full PDFs and analyzes key sections (intro, methodology, results, conclusion).
- Pairwise comparisons are performed by GPT-5.2 to determine which paper has a higher scientific impact.
- Live tournament progress tracking, including expandable logs showing the LLM's reasoning for each match.
- Final results are displayed as a ranked list using Bradley-Terry scores.
- History page to view past and ongoing tournaments.
- Performance optimizations are in place, including MongoDB indexing and lazy-loading of data to ensure fast page loads.
- A critical reliability feature automatically resumes any tournaments that were interrupted by a server restart.
