# PaperSumo by Kurate.org — Scientific Preprint Ranking System

## Problem Statement
Build a robust system for ranking and validating AI model performance on scientific preprints. Features a leaderboard tournament where different LLMs act as judges to rank papers, with validation against human peer-review ground truth.

## Architecture
- **Frontend**: React + Shadcn UI (production build via `serve`)
- **Backend**: FastAPI + MongoDB
- **LLMs**: GPT-5.2, Claude Opus 4.6, Gemini 3 Pro (via Emergent LLM key)
- **Domain**: kurate.org

## Optimal Configuration (as of Mar 2 2026)
- **Summarizer**: Opus 4.6 Thinking (used in live tournaments; GPT/Gemini summaries generated for analysis only)
- **Judges**: Round-robin across GPT-5.2, Opus 4.6, Gemini 3 Pro (HIGH confidence this beats any single judge by +0.05-0.15 rho)
- **Input format**: Abstract + AI impact assessment summary
- **Pair selection**: Cross-tier only (same-tier filtered out)
- **Summary source**: "claude" (only Claude Thinking summaries used in live tournaments)

## Key Findings

### Summarizer Comparison (CORRECTED — same-pair, Mar 1)
Previous analysis was biased by pair selection. Corrected same-pair results (243 pairs, iclr-llm + iclr-codegen):
- Opus 4.6 Thinking: rho=0.643, acc=81.1%
- Opus 4.5: rho=0.624, acc=76.5%
- Opus 4.6: rho=0.584, acc=78.6%
- Gemini 3 Pro: rho=0.570, acc=75.7%
- GPT-5.2: rho=0.556, acc=76.1%
- **Gap is only 0.087 rho** — earlier biased analysis showed 0.336

### Round-Robin vs Single Judge
HIGH confidence finding. On every summarizer and every dataset, round-robin beats single judge:
- Diversity benefit: +0.05-0.15 rho
- Even the worst summarizer (Gemini) + round-robin beats the best summarizer + single judge

### Intransitive Cycles
- Format-normalized (same-pair): Gemini 1.03%, Opus 4.5 1.34%, GPT 1.66%
- Self-consistency bias REJECTED: models judging own summaries have MORE cycles

## CRITICAL METHODOLOGY NOTE
**Always compare on exact same pairs.** Multiple findings were initially wrong due to pair-selection bias.

## elife-comp-sys-bio Investigation (Mar 2 2026)
**Finding: The data is not corrupt. The "summaries" field is empty but summaries exist at the top level.**

- 80 papers, all with full_text
- `summaries` dict: EMPTY (0/80 papers)
- `ai_impact_summary`: EXISTS (generated by claude-opus-4-6-thinking, ~7500 chars each)
- 1,197 matches with content_mode=abstract_plus_summary (correctly used ai_impact_summary field)
- Abstracts contain HTML-formatted eLife editorial assessment (sig/strength labels)
- Human scores available as sig_score (significance 1-5) and str_score (strength 1-6)

**What needs attention:**
1. Papers have `ai_impact_summary` at top level but NOT in `summaries` dict — means they won't participate in the live leaderboard tournament (which checks for `summaries` dict)
2. Abstracts include eLife editorial assessments — AI judges can see the human verdict, which could inflate correlation artificially
3. All elife datasets (cancer, microbiology, neuro-100) have the same pattern: 0 in `summaries` dict
